# 🏙️ QuickSnippet Pro: The Secure Cyber-Baroque Vault

**QuickSnippet Pro** — это промышленный инструмент для безопасного хранения знаний с использованием криптографически изолированных сегментов данных. Система спроектирована по принципу «Zero-Trust», где ни один бит информации не попадает в БД без предварительной стерилизации и шифрования.

## 🛠 Архитектурный Стек

* **Backend:** FastAPI (Python 3.13) с асинхронным ядром.
* **Database & Cache:** PostgreSQL (SQLModel/SQLAlchemy) + Redis (Cache-Aside pattern).
* **Frontend:** Гипер-легковесный стек HTMX + Alpine.js. Полный отказ от тяжелых SPA-фреймворков.
* **Security Kernel:** Кастомное крипто-ядро (Argon2id для паролей, AES-256-GCM для данных).
* **Client:** PWA-ready архитектура с поддержкой API (Bearer Token) для десктоп/mobile клиентов на Flet.

## 🛡️ Инженерные особенности (Security Engineering)

* **Authenticated Encryption:** Использование `AES-256-GCM` гарантирует не только скрытность, но и защиту от подмены (Tamper-proof).
* **Category-based Key Shifting:** Ключи шифрования сегментированы по категориям (`Code`, `Study`, `Ideas`). Компрометация одного сегмента не раскрывает остальные.
* **Security Breach Lockdown:** Встроенный `SecurityAuditMiddleware` мгновенно инвалидирует сессию при ошибке дешифровки, предотвращая попытки инъекций в крипто-контейнеры.
* **Smart Sanitization:** Входящие данные проходят через Rust-написанный санитайзер `nh3` до этапа обработки ORM.

## ⚡ Производительность & UX

* **Cache-Aside Pattern:** Использование Redis позволяет отдавать часто запрашиваемые данные с нулевой задержкой, минимизируя нагрузку на Postgres.
* **Intelligent Auto-Tagging:** Встроенный анализ через `Pygments` автоматически определяет синтаксис кода, позволяя системе самостоятельно задавать параметры подсветки.
* **Neon-Syntax Highlighting:** Интеграция `Prism.js` с динамической подгрузкой языков через `Autoloader`.

## 🔗 Nexus Links: Реляционный граф знаний

Система поддерживает иерархическую структуру `parent_snippet_id`. При удалении узла, каскадные связи автоматически перестраиваются (принцип `SET NULL`), сохраняя целостность базы знаний без потери связанных фрагментов.
