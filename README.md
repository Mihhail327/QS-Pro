# 🏙️ QuickSnippet Pro: The Secure Cyber-Baroque Vault

**QuickSnippet Pro** — это промышленная local-first платформа для безопасного хранения знаний с использованием криптографически изолированных сегментов данных. Система спроектирована по принципу **Zero-Trust**, где ни один бит информации не попадает на сервер без предварительной стерилизации, а на стороне клиента развернута автономная PWA-инфраструктура.

## 🛠 Архитектурный Стек

* **Backend:** FastAPI (Python 3.12+) с полностью асинхронным ядром.
* **Database & Cache:** PostgreSQL (SQLModel/SQLAlchemy) + Redis (Cache-Aside pattern).
* **Frontend:** Декларативный стек **HTMX + Alpine.js** с нулевым оверхедом. Полный отказ от SPA-фреймворков.
* **Observability:** Распределенный трейсинг **OpenTelemetry** и структурное JSON-логирование через **structlog**.
* **PWA & Offline:** Автономный движок синхронизации на базе **Service Worker + IndexedDB**.
* **Client App:** Кроссплатформенный клиент на **Flet** (Python) для Desktop и Mobile.

---

## 🛡️ Инженерная Безопасность (Security Engineering)

* **Authenticated Encryption:** Шифрование данных по стандарту `AES-256-GCM` гарантирует конфиденциальность и встроенную защиту от подмены данных (Tamper-proof).
* **Category-based Key Shifting:** Криптографические ключи сегментированы по категориям (`Code`, `Study`, `Ideas`). Компрометация одного сегмента оставляет остальные данные в безопасности.
* **Zero-Trust Authentication:** Регистрация и авторизация на базе современного хэширования **Argon2id** (замена устаревшему PBKDF2) со встроенным анализатором сложности паролей.
* **Security Breach Lockdown:** Асинхронный `SecurityAuditMiddleware` мгновенно инвалидирует сессию пользователя при любой ошибке дешифровки, блокируя brute-force и инъекции в крипто-контейнеры.
* **Smart Sanitization:** Входящие данные проходят очистку через Rust-бинд `nh3` до этапа обработки ORM, исключая XSS-векторы. Стерилизация изображений через `Pillow` (преобразование в WebP с полным выжиганием EXIF/GPS метаданных).

## ⚡ Особенности Реализации & UX

* **True Local-First (Offline Sync):** Модернизированный Service Worker перехватывает запросы в оффлайне, извлекая данные из локального `IndexedDB`. Очередь транзакций (`sync-queue`) кэширует операции создания/удаления без интернета и автоматически синхронизирует их с PostgreSQL при восстановлении сети.
* **Мультиязычность (i18n):** Динамическое переключение интерфейса (RU, EN, ET) на стороне Alpine.js с сохранением состояния в `localStorage` без перезагрузки бэкенда.
* **Промышленный Мониторинг:** Инжекция сквозного `trace_id` и `span_id` во все логи (`uvicorn`, `fastapi`, кастомные сервисы). Логи автоматически разделяются: цветной `ConsoleRenderer` для локальной разработки и строгий структурированный JSON для систем сбора логов (Grafana Loki/Jaeger) в продакшене.
* **Защита мобильного ввода:** Принудительное отключение автокапитализации (`autocapitalize="none"`, `ft.TextCapitalization.NONE`) и подсказок на полях ввода логина/пароля для предотвращения искажения учетных данных мобильными ОС.
* **Nexus Links (Реляционный граф знаний):** Поддержка иерархических связей `parent_snippet_id`. При удалении родительского узла связи автоматически перестраиваются по принципу `SET NULL`, сохраняя целостность фрагментов.
